# v0.9.1 Copilot audit backlog

**Source:** GitHub Copilot full-codebase audit, 2026-04-20 (52 findings тАФ 6
critical, 10 high, 13 medium, 8 low, 10 test-gap, 5 architecture). 13 items
shipped pre-tag in the v0.9.0 hotfix bundle (`feature/hotfix-v0.9.0-copilot-critical`).
This doc tracks the remaining 39.

**Status:** Tracked, not yet scheduled. Each item becomes a Kimi loop in
sequence (or is resolved as "won't fix" with a documented reason). Ordering
below is the suggested work order; Cursor may cluster related items into a
single loop when the diff stays small.

**Merge discipline:** Per `.cursorrules` "Post-release merge discipline,"
loops here do **not** merge to `develop` until a `v0.9.1-release-prep.md`
document names them by exact `feature/*` branch. Thematic grouping on this
backlog page is **not** authorization. This file exists to schedule work,
not to authorize merges.

**Already shipped in v0.9.0 (for reference, do not repeat):** C-1, C-2, C-3,
C-4, C-5, C-6, H-1, H-2, H-3, H-4, H-9, A-3, T-1. See CHANGELOG `[0.9.0]`
"Pre-release hotfixes" subsection for the per-item detail.

## Queue

### Priority 1 тАФ remaining High severity (5 items)

- **H-5** ┬╖ `src/hestia/config.py` ┬╖ `model.name == "dummy"` is silently
  accepted and produces mystifying runtime errors later. Raise `ValueError`
  at config-load when the model name is the literal `"dummy"` *unless*
  `HESTIA_ALLOW_DUMMY_MODEL=1` (for the CI/test path that currently relies
  on it). Update the fixtures that use "dummy" to set the env var or switch
  to a real-but-unreachable URL.
- **H-6** ┬╖ `src/hestia/policy/default.py:186-189` ┬╖ per-call
  `logging.getLogger(__name__)` inside `_decide_retry`. Move the logger to
  module level. Trivial cleanup; batch with H-8 and any other single-line
  lints from the same file.
- **H-7** ┬╖ `src/hestia/tools/builtin/delegate_task.py:158` ┬╖
  `artifact_refs=[]` is hard-coded in the returned `ToolCallResult`. The
  delegated turn already produces artifacts; extract them from the
  delegated turn result (`turn.artifacts` or equivalent) and pass through.
  Add a regression test that delegates a tool call producing an artifact
  and asserts the parent's `ToolCallResult.artifact_refs` is non-empty.
- **H-8** ┬╖ `src/hestia/app.py:224` ┬╖ `confirm_callback: Any = None`.
  Tighten to `ConfirmCallback | None` (the type alias already exists in
  `src/hestia/platforms/types.py` or equivalent тАФ confirm its home before
  importing). If a runtime shape check is needed, add `isinstance` in
  `set_failure_handler`-style mutator.
- **H-10** ┬╖ `src/hestia/persistence/trace_store.py` vs
  `src/hestia/persistence/schema.py` ┬╖ duplicate `traces` DDL. `trace_store`
  currently executes inline `CREATE TABLE` on init; schema.py owns the
  canonical shape. Consolidate: remove the inline DDL, have `trace_store`
  rely on the Alembic-managed schema, add a startup assert that the table
  exists. Pairs with A-2 (DDL fragmentation) тАФ may be addressed together.

### Priority 2 тАФ Medium severity (13 items)

- **M-1** ┬╖ `src/hestia/memory/store.py:61` ┬╖ bare `except` on the FTS5
  capability probe swallows `DatabaseError` and similar. Narrow to
  `sqlite3.OperationalError` + `aiosqlite.OperationalError` or log the
  unexpected exception at `ERROR` before falling through.
- **M-2** ┬╖ `src/hestia/orchestrator/engine.py:352` ┬╖ history is re-fetched
  from `SessionStore` on every iteration of the tool-call loop. Keep the
  in-memory list updated across iterations and only re-fetch on initial
  turn entry (or never, if context is already authoritative in memory).
  Add a benchmark assertion: N-iteration turn should hit the store at most
  once for message history.
- **M-3** ┬╖ `src/hestia/context/builder.py:338` ┬╖ `_count_tokens` is
  awaited sequentially in a loop. Use `asyncio.gather` to parallelize. Add
  a cap (e.g. `asyncio.Semaphore(8)`) if downstream tokenizer service has
  known concurrency limits.
- **M-4** ┬╖ `src/hestia/security/injection.py:56` ┬╖ the regex for prompt
  injection markers false-positives on YAML/JSON config blocks and any
  text that mentions "system:" in quoted form. Add a length cap (skip
  strings under e.g. 40 chars) and tighten the anchor so it requires a
  preceding newline or start-of-string.
- **M-5** ┬╖ `src/hestia/identity/compiler.py:152` ┬╖ `_check_cache(source_hash)`
  is defined but never called. Remove and simplify surrounding flow.
- **M-6** ┬╖ `src/hestia/commands.py:751` and
  `src/hestia/policy/default.py:134` ┬╖ the `0.85` magic number
  (context-pressure threshold) is duplicated. Extract to a shared constant
  in `hestia.policy` (e.g. `CONTEXT_PRESSURE_THRESHOLD = 0.85`) and import.
- **M-7** ┬╖ `src/hestia/email/adapter.py:247` ┬╖ IMAP UID order is assumed
  chronological. IMAP does not guarantee this (server-side). Sort the
  returned envelope list by `INTERNALDATE` before returning.
- **M-8** ┬╖ `src/hestia/app.py:316` ┬╖ private attribute
  (`app._failure_handler` or similar) is set from outside the class.
  Expose a public `set_failure_handler()` method on `CliAppContext` and
  route callers through it.
- **M-9** ┬╖ `src/hestia/config.py:428-438` ┬╖ `from_file` executes arbitrary
  Python (the config is a `.py` file with top-level statements). This is
  **intentional** тАФ Hestia is a single-operator tool тАФ but not documented.
  Resolve by adding a `SECURITY.md` note (and a docstring reference in
  `from_file`) explicitly calling out the single-operator assumption.
  **Won't-fix as code change.**
- **M-10** ┬╖ `src/hestia/config.py:73` ┬╖ `allowed_roots` defaults to
  `["."]` (current working directory). In a `systemd --user` deployment
  the CWD is often `$HOME` or `/`, which is wider than intended. Keep the
  default but have `hestia doctor` warn when `allowed_roots` contains `"."`
  and the resolved CWD is `$HOME` or `/`.
- **M-11** ┬╖ `src/hestia/tools/builtin/delegate_task.py:155` ┬╖ a new
  `Orchestrator` is constructed per delegation, which means no
  SlotManager/inference-slot sharing with the parent. For now add a
  comment documenting the trade-off (isolation vs VRAM pressure). A real
  fix тАФ thread the parent's SlotManager through тАФ is larger and lives in
  Priority 5 (Architecture).
- **M-12** ┬╖ `src/hestia/policy/default.py:103` ┬╖ "subagent" is compared
  as a string literal. Promote to a `PlatformName` `Literal` or `Enum`
  value and reuse across the policy engine.
- **M-13** ┬╖ `src/hestia/memory/store.py:330` ┬╖ WHERE clause is assembled
  with an f-string (not parameter-bound). Inputs are internal (no user
  injection vector today), but switch to `sqlalchemy.sql.expression`
  param-binding for consistency with the rest of the store.
