## Track 3 — Pre-tag hotfix bundle: Copilot critical + selected high findings

**Branch:** `feature/hotfix-v0.9.0-copilot-critical`. Cursor may split into two or three feature branches if the step budget requires, but all must land before Track 4 tags.

**Scope:** 13 findings. Criteria for inclusion: data corruption, security, event-loop stalls, or easy-to-fix crashes that hit during normal use. Everything else deferred to Track 5.

### Critical (6 — all 6 land pre-tag)

- **C-1.** `src/hestia/persistence/trace_store.py:247` — replace `datetime.now().isoformat()` with `utcnow().isoformat()`. Scan the rest of the file for any other naive-datetime usage and fix in the same commit. All other timestamps in the codebase are UTC; this one is local-naive and corrupts time-range joins.
- **C-2.** `src/hestia/persistence/sessions.py` — `append_message` (~L255-297) and `append_transition` (~L475). Both do `SELECT MAX(idx) + 1` followed by `INSERT` in separate connections. Two concurrent callers on the same session produce primary-key collisions. Wrap SELECT+INSERT in a single transaction via `async with conn.begin()`. If SQLite serialization breaks, use `INSERT OR IGNORE ... SELECT MAX(idx) + 1 FROM messages WHERE session_id = :sid` pattern and retry-on-collision. Add regression test `tests/unit/test_append_message_race.py` that spawns 20 concurrent `append_message` calls for the same session and asserts 20 distinct `idx` values with no `IntegrityError`.
- **C-3.** `src/hestia/artifacts/store.py:65-68, 74-77, 158, 184, 217` — five synchronous file I/O sites inside `async` methods. Wrap each with `await asyncio.to_thread(...)`. `__init__` is the trickiest (it currently does sync mkdir + json.load at construction); convert to a classmethod `async def open(root: Path) -> ArtifactStore` factory and deprecate direct construction. Document in the class docstring that direct `__init__` is for tests/sync use only.
- **C-4.** `src/hestia/orchestrator/engine.py:398, 404` — define `MaxIterationsError(HestiaError)` and `PolicyFailureError(HestiaError)` in `src/hestia/errors.py`. Replace `raise Exception(...)` at those two sites with the new types. Verify `classify_error` in whichever module owns it now handles both; if not, add explicit cases.
- **C-5.** `src/hestia/doctor.py:107-113, 217` — `_check_dependencies_in_sync` is `async def` but calls blocking `subprocess.run` and `httpx.get`. Either: (a) swap to `asyncio.create_subprocess_exec` + `httpx.AsyncClient`, or (b) wrap both in `asyncio.to_thread`. Prefer (a) for subprocess; (b) is acceptable for the one `httpx.get` call. Add a `# async-safe` assertion or test that the function can be called concurrently without stalling the event loop.
- **C-6.** `src/hestia/email/adapter.py:105-110` — `smtp.starttls()` return is ignored. Either: assert `(220, *)` return code and raise a `EmailConfigError` otherwise, OR migrate to `smtplib.SMTP_SSL` on port 465 and remove the STARTTLS dance entirely. Prefer SMTP_SSL unless there's a known provider incompatibility. Document the decision in the docstring.

### High (5 of 10 land pre-tag; remaining 5 in Track 5)

- **H-1.** `src/hestia/core/inference.py:201` — `data["choices"][0]` crashes on empty array. Guard with:
  ```python
  choices = data.get("choices", [])
  if not choices:
      raise InferenceServerError("inference returned no choices")
  choice = choices[0]
  ```
- **H-2.** `src/hestia/core/inference.py:214` — after `json.loads(fn["arguments"])`, add `if not isinstance(arguments, dict): raise InferenceServerError(f"tool_call arguments not a dict: {type(arguments).__name__}")`. Otherwise `**arguments` throws `TypeError` later with unclear provenance.
- **H-3.** `src/hestia/tools/builtin/write_file.py:40` — add `encoding="utf-8"` to the `write_text` call. Grep the rest of `src/hestia/tools/builtin/` for other `write_text` and `read_text` without explicit encoding and fix any found in the same commit.
- **H-4.** `src/hestia/tools/builtin/web_search.py:36-38` — move Tavily API key from request body to `Authorization: Bearer <key>` header. Verify Tavily still accepts the bearer form (it does, per their docs); if not, fall back to the body but document the trade-off.
- **H-9.** `src/hestia/tools/registry.py:109-115` — current tool-execution wrapper only catches `(TypeError, ValueError, OSError)`. `RuntimeError`, `httpx.HTTPError`, and async cancellations escape. Widen to a documented exception contract: create `ToolExecutionError(HestiaError)` that wraps the underlying exception; catch `Exception` (not `BaseException` — preserve `CancelledError`), wrap and re-raise. Add the exception type to the tool-result JSON so the orchestrator can classify it without string-matching.

### Architecture (1 of 5 lands pre-tag)

- **A-3.** `src/hestia/policy/engine.py` `PolicyEngine` ABC does not declare `ctx_window`, but `src/hestia/commands.py:761` and other callsites access `policy.ctx_window` directly. A custom `PolicyEngine` subclass that doesn't inherit `DefaultPolicyEngine` will `AttributeError` at runtime. Add `ctx_window: int` as an abstract property (or required `__init__` parameter stored as public attribute) to the ABC; update `DefaultPolicyEngine` to override as already written. Add a test that constructs a minimal custom `PolicyEngine` subclass and verifies `ctx_window` is enforced.

### Test gap (1 of 10 lands pre-tag)

- **T-1.** `tests/unit/test_append_message_race.py` — bundled with C-2, above.

**Scope discipline:** 13 items only. Do not pull H-5, H-6, H-7, H-8, H-10 into this branch even though they look easy — they're explicitly triaged to v0.9.1 and pulling them in defeats the "only ship what blocks a release" principle.
