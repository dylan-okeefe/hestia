# Handoffs

Handoff documents capture what changed in a given development loop and why. They are written at loop completion, reviewed before merge, and serve as the historical record of how Hestia evolved.

Each handoff corresponds to one numbered loop (L01, L02, …). Files are named `L<number>-<topic>-handoff.md` and are ordered chronologically by loop number.

If you are looking for current architecture or operating guidance, see the [ADR index](../adr/) and [guides](../guides/) instead.
